## Быстрый старт

### Шаг 0 — Положить входные файлы

```
data/
├── transactions.csv   ← выдаётся отдельно
├── gold_set.csv       ← выдаётся отдельно
└── categories.json    ← выдаётся отдельно
```

### Шаг 1 — Python (очистка + БД + ноутбук)

```bash
pip install -r requirements.txt
python load_db.py          # очистка CSV → SQLite (transactions.db)
python generate_mocks.py   # JSON-моки для фронтенда → public/data/
jupyter notebook notebooks/analysis.ipynb
```

### Шаг 2 — Frontend

```bash
npm install
npm run dev
# Открыть http://localhost:3000
```

> **Важно:** `generate_mocks.py` должен отработать до `npm run dev`,  
> иначе `public/data/` будет пустым и таблица не загрузится.

## Краткие выводы (Часть 1)

1. **80 800 строк**, период **01.01.2024 — 30.04.2026**, типы: INVOICE / WAYBILL / ACT.
2. **800 дубликатов** удалено (~1%); правило: совпадение sender+receiver+date+amount+description.
3. **Валидных БИН/ИИН:** 91.0% отправителей, 91.2% получателей — ~7 200 записей не проходят алгоритм РК.
4. **847 пропусков** в `description` заменены на «Нет описания».
5. **Топ-1 пара** по обороту: `960819662090 → 711020691210` — ₸298.9 млн, 107 операций.
6. **Аномалия 1:** 380 транзакций с `amount_kzt < 0` — сторнировки или ошибки ввода.
7. **Аномалия 2:** 1 само-перевод (`sender_id == receiver_id`) — невалидная операция.
8. **Аномалия 3:** 7 541 транзакция превышает 3×IQR; максимум ₸14 998 620.
9. **15 контрагентов** получают >70% входящих от единственного источника — признак концентрации.
10. Помесячная динамика стабильна (~₸5–6.5 млрд/мес), без выраженной сезонности.

## Frontend (Ветка C) — что реализовано

- **Таблица контрагентов** — пагинация (15/стр.), сортировка по 4 полям, фильтры по дате и типу документа
- **Карточка контрагента** — 4 вкладки: Обзор / Динамика / Документы / Партнёры
- **Графики:** Area, Bar, Pie chart (помесячная динамика, распределение по документам, категориям)
- **Граф партнёров** — Canvas + force-directed layout, hover показывает суммы на рёбрах
- **Адаптивная вёрстка** — без горизонтального скролла на 360px
